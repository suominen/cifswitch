CIFSwitch is a Linux kernel local privilege escalation in the CIFS
client: the kernel never validated that a `cifs.spnego` key description
originated from kernel code, so an unprivileged user can forge one and
steer the rootful `cifs.upcall` helper (cifs-utils >= 6.14) into loading
an attacker-controlled NSS module as root. The canonical fix is Linus
mainline commit 3da1fdf4efbc490041eb4f836bf596201203f8f2, which adds a
`.vet_description = cifs_spnego_key_vet_description` hook in
fs/smb/client/cifs_spnego.c (pre-6.7 path: fs/cifs/cifs_spnego.c). No CVE
has been assigned yet — the tracker uses the placeholder CVE-2026-XXXXX.

References:
- Writeup: https://heyitsas.im/posts/cifswitch/
- Public PoC: https://github.com/manizada/CIFSwitch
- Kernel fix: https://github.com/torvalds/linux/commit/3da1fdf4efbc490041eb4f836bf596201203f8f2

Working-tree hygiene: the wrapper refuses to run on a dirty tree
(`git status --porcelain`, which counts untracked files too), so a stray
file left in the worktree blocks every subsequent scheduled run. Stream
any fetch through a pipe — `curl -fsSL <url> | grep …` — and never save a
download into the worktree. If you genuinely need a scratch file, create
it under a `mktemp -d` directory and delete it before you finish; the only
thing you may leave behind is the edited tracker (and its commit).
